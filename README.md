# StablecoinRatings

**The independent risk layer for stablecoins.**

Professional-grade stablecoin evaluation built on transparent, observable data. Every score is derived from free public APIs and clearly defined criteria. No black boxes. No subjective overrides.

---

## Quick Start

### View the site

The live site is at **https://stablecoinratings.org**

Data refreshes daily at 02:00 UTC via a systemd timer on the VPS.

### Run locally

```bash
# Clone the repo
git clone https://github.com/rurogge/stablecoinratings_website
cd stablecoinratings_website

# Install dependencies
pip install -r requirements.txt

# Run the data pipeline
python3 pipeline/pipeline.py

# Serve locally (any static server)
python3 -m http.server 8000 --directory site
# → Open http://localhost:8000
```

## Deployment

### VPS Setup (systemd — recommended)

```bash
# Copy pipeline runner
sudo cp pipeline/run_pipeline.sh /usr/local/bin/stablecoinratings-runner
sudo chmod +x /usr/local/bin/stablecoinratings-runner

# Install systemd units
sudo cp systemd/stablecoinratings.service /etc/systemd/system/
sudo cp systemd/stablecoinratings.timer /etc/systemd/system/
sudo systemctl daemon-reload

# Enable and start the timer
sudo systemctl enable stablecoinratings.timer
sudo systemctl start stablecoinratings.timer

# Check status
sudo systemctl status stablecoinratings.timer
journalctl -u stablecoinratings.timer --no-pager
```

The pipeline runs daily at 02:00 UTC with up to 15 minutes randomized delay to avoid API thundering herd.

### Cloudflare Pages Setup

1. Push this repo to GitHub (see instructions below)
2. Go to https://pages.cloudflare.com → Create a project
3. Connect your GitHub repo
4. Set:
   - **Project name:** `stablecoinratings`
   - **Production branch:** `main`
   - **Build command:** (leave empty — pure static)
   - **Build output directory:** `site`
5. Add secrets to GitHub repo → Settings → Secrets:
   - `CLOUDFLARE_API_TOKEN` — from Cloudflare dashboard → Profile → API Tokens
   - `CLOUDFLARE_ACCOUNT_ID` — from Cloudflare dashboard → Workers & Pages → overview
   - `COINGECKO_API_KEY` — (optional) from CoinGecko — improves rate limits
   - `ETHERSCAN_API_KEY` — (optional) from Etherscan — improves contract verification

The GitHub Actions workflow runs daily at 02:30 UTC and deploys automatically.

## Push to GitHub

On this VPS, run:

```bash
cd /root/stablecoinratings

# Initialize git (if not already done)
git init
git remote add origin https://github.com/rurogge/stablecoinratings_website.git

# Configure git (replace with your name/email)
git config user.name "Ruggero"
git config user.email "ncprugger0@namecheap.com"

# Add all files (data/ is tracked — it contains the initial seed data)
git add .
git commit -m "Initial commit: full stablecoin ratings site + pipeline + data"

# Push to GitHub (will prompt for credentials)
git push -u origin main
```

To create a Personal Access Token (PAT):
→ https://github.com/settings/tokens → "Fine-grained tokens" → Generate new token
Scopes needed: **Contents** (read/write), **Pull requests** (read/write), **Deployments** (read/write)

After pushing, go to **https://pages.cloudflare.com** → your project → Settings → Environment variables, and add:
- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

The GitHub Actions workflow (`deploy.yml`) will then run daily and deploy automatically.

### Cloudflare Pages — Enable Clean URLs

After first deploy, go to your Cloudflare Pages project → Settings → Build configuration:
- Toggle **"Serve Pages without file extension"** → ON

This makes `/methodology`, `/history`, `/data` work identically to `/methodology.html`, etc.

## Open Data

All ratings data is freely available as JSON — no API key required:

| File | URL | Description |
|------|-----|-------------|
| Full ratings | `/data/ratings_full.json` | All pillars, all coins, sparklines |
| Grades | `/data/grades.json` | Compact coin → grade lookup |
| History | `/data/history.json` | 365-day rolling archive |

Example usage:

```javascript
// Fetch latest grades
const res = await fetch('https://stablecoinratings.org/data/grades.json');
const { coins } = await res.json();
for (const [symbol, info] of Object.entries(coins)) {
  console.log(`${symbol}: ${info.grade} (${info.score})`);
}
```

## Methodology

See [Methodology](https://stablecoinratings.org/methodology) for the full framework documentation.

Six pillars — each scored 0–100:
1. **Peg Stability (30%)** — price deviation, depeg history, recovery speed
2. **Reserve Quality (25%)** — composition, attestation, bankruptcy remote
3. **Liquidity (15%)** — volume/mcap, bid-ask spread, DEX TVL
4. **Management (15%)** — regulatory status, enforcement history, team track record
5. **Smart Contract Security (10%)** — audit status, exploit history, upgradeability
6. **Decentralization (5%)** — chain diversity, minting permission, freeze controls

|-----|-------------|---------|
| CoinGecko | coingecko.com/api | Higher rate limits; more price history |
| Etherscan | etherscan.io/apis | Smart contract audit verification |

Set keys as environment variables or in `~/.hermes/.env` on the VPS.
